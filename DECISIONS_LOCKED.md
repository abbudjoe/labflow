## Domain workflows

### Workflow A — DNA quantification

- Analyte: dsDNA.
- Assay: Quant-iT PicoGreen.
- Instrument: Varioskan.
- Output: TSV.
- Standards: 8 standards run once per batch on separate standards plate.
- Standards location: A1-H1.
- Sample plates: 96-well with 95 samples + 1 blank.
- Blanks: one blank well on each sample plate.
- Curve model: linear.
- Dilution factor required.
- Out-of-range handling: dilute and rerun.
- Historical throughput claim: 12 x 96 samples in under 1 hour, including plate setup, reader time, standard curve, LIMS upload, and QC review. No tracked speedup percent.

### Workflow B — DNA normalization

- Analyte: dsDNA.
- Source concentration unit: ng/µL.
- Target mode: final concentration + final volume, or final mass + final volume.
- Molarity is out of scope.
- Final mass is derived from concentration x volume when concentration + volume are provided.

### Workflow C — RNA normalization + re-quantification

- Analyte: total RNA.
- Upstream post-extraction quant may have used Qubit RNA or Nanodrop.
- Normalization/sample-prep quant/re-quant used RiboGreen.
- Re-quant performed on normalized RNA or intermediate RNA dilution plate.
- Re-quant value becomes downstream concentration.
- No default percent tolerance.
- Missing/invalid re-quant results require manual review or repeat quant.
- Throughput improvement: approximately 3x by moving from single-container batches to 3-container batches, enabled by LIMS batch automation and reduced robot idle time.

## Labware/container rules

- Plate coordinate system: rows A-H, columns 1-12.
- Historical container family: Thermo Fisher Matrix 96 x 1 mL tube rack.
- DNA closure: rubber/septum top.
- RNA closure: screw top.
- Max destination tube working volume: 999 µL.
- Container type should be resolved from LIMS/container registry, not hard-coded everywhere.

## Liquid handling rules

- Minimum transfer volume: 1 µL.
- Source residual dead volume: 2 µL.
- Robot aspiration safety margin: 1 µL.

Required source volume:

```text
required_source_volume_ul =
    sample_transfer_volume_ul
    + source_residual_dead_volume_ul
    + robot_aspiration_safety_margin_ul
```

## Normalization modes

### Standard new-container normalization

- Destination container exists.
- Diluent first.
- Sample second.
- Mix after sample addition.
- JANUS receives CSV generated by LIMS showing well, diluent volume, sample volume.

### In-place normalization

- No destination container.
- Used if available sample volume is less than or equal to transfer volume required for standard normalization.
- Only diluent is added.
- No sample transfer occurs.
- Mixing is skipped.

### Split workflow

- Trigger: calculated sample transfer volume < 1 µL because concentration is high.
- Use 1 µL source sample.
- Create first diluted child sample.
- Re-quant child sample.
- Follow-up normalization uses measured child concentration.

## Batch model

- Samples per sample plate: 95.
- Blank wells per sample plate: 1.
- Standards plate per batch: true.
- Standards count: 8.
- Standards wells: A1-H1.
- RNA optimized containers per batch: 3.
- Robot run time per container: 2-4 minutes.
- Human prep time per batch: ~10 minutes.

## LIMS invariants

- Sample ID required.
- Sample ancestry tracked by sample ID.
- Source location required.
- Destination location required for valid standard/new-container samples.
- Duplicate sample ID blocks.
- Duplicate source well blocks.
- Duplicate destination well blocks.
- Every exception is reported.
- Invalid samples generate no robot transfer.
- Every manual override is audited.

## AI rules

- LLMs can explain, retrieve, propose, summarize, and call tools.
- LLMs cannot bypass deterministic validation.
- LLMs cannot generate robot-ready artifacts for invalid batches.
- State-changing actions require dry-run, approval, and audit.
- Answers must cite retrieved sources or say not supported.
