# Demo Audit Report

This report summarizes the synthetic audit evidence generated by the LabFlow
demo. It is intentionally human-readable so portfolio reviewers can inspect the
control path without opening raw JSONL logs.

## Scope

- Demo workflow: RNA normalization and re-quantification.
- Data type: synthetic only.
- Commit actions: none.
- Robot execution: none.
- JANUS-style files: dry-run previews only.

## Audit Chain

| Step | Tool | Mode | Workflow | Result | Key Evidence |
| ---: | --- | --- | --- | --- | --- |
| 1 | `validate_batch` | `read_only` | `RNA_BATCH_BAD_001` | `invalid` | Missing blank, missing concentration, invalid standard curve/well, JANUS blocked. |
| 2 | `validate_batch` | `read_only` | `RNA_BATCH_FIXED_001` | `valid` | Fixed workflow passes deterministic validation. |
| 3 | `generate_janus_csv` | `dry_run` | `RNA_BATCH_BAD_001` | `blocked` | Invalid batch does not produce robot-facing transfers. |
| 4 | `generate_janus_csv` | `dry_run` | `RNA_BATCH_FIXED_001` | `ok` | Valid batch produces a JANUS-style preview artifact. |
| 5 | `generate_normalization_plan` | `read_only` | `RNA_BATCH_FIXED_001` | `valid` | Normalization plan remains deterministic and read-only. |
| 6 | `parse_varioskan_tsv` | `read_only` | synthetic TSV inputs | `ok` | Instrument-style TSV parsing is audited as read-only evidence. |

## Guardrail Evidence

- Invalid workflows are validated before artifact generation.
- Missing concentrations remain blocking facts; the system does not invent them.
- The invalid JANUS dry-run is blocked with `JANUS_BLOCKED_FOR_INVALID_BATCH`.
- The fixed JANUS output is still a dry-run preview, not a robot command.
- No approval token appears in this demo because no commit action is performed.

## Reviewer Notes

The ignored raw file `examples/expected/audit_log.jsonl` can be regenerated
locally by running:

```sh
python3 scripts/run_demo.py
```

For the public portfolio path, this markdown report is the canonical audit
evidence.
